# AutoResearch 自动研究闭环

实践 Karpathy 的 **generate–test–learn** 理念：把"研究策略能否盈利"这一人类科研过程，编码成一个**自动闭环**——系统自己生成假设、用历史数据测试、从真实表现中学习，并持续反哺下一轮优化，最终提炼出**跨股票、跨时期都稳定盈利**的交易策略，作为平台选股的核心方案。

## 闭环架构

```
┌─────────────────────────────────────────────────────────────────┐
│  1. Generate      2. Test          3. Aggregate    4. Recommend │
│  参数网格采样  →  Walk-Forward  →  全局中位数   →  多策略共识   │
│  (笛卡尔积)       回测(60/40)      聚合提炼         (可信度加权) │
└─────────────────────────────────────────────────────────────────┘
         ▲                                                        │
         │                                                        ▼
┌─────────────────────────────────────────────────────────────────┐
│  6. Learn                        5. Resolve                       │
│  贝叶斯可信度融合             ←  真实收益结算                     │
│  (先验回测+后验真实)             (止盈/止损/过期)                  │
│  反哺下一轮搜索与推荐权重                                         │
└─────────────────────────────────────────────────────────────────┘
```

每一环都是自动、可重复、可验证的。`POST /auto-cycle` 一键串起 3→4→5→6；`POST /optimize` 触发 1→2 并在完成后自动衔接 3→6。

## 核心机制

### 防过拟合（Test 环）
- **Walk-forward 切分**：每只股票前 60% 训练、后 40% 测试（测试段回退预热期覆盖指标）。
- **调和均值评分**：`composite = 2·train·test/(train+test)`。train 极好但 test 极差时，调和均值趋近较小值，自动拒绝过拟合方案（远优于算术均值）。
- **过拟合度门槛**：`overfit = |train−test|/max(train,test) > 0.6` 直接拒绝。
- **交易笔数置信度**：测试段交易数 < 5 笔时评分打折（统计意义不足），并排除出全局稳健聚合。
- **样本外盈利硬门槛**：最优方案必须测试期真实盈利才入选。

### 全局稳健策略提炼（Aggregate 环 ——「最稳定策略」）
单只股票的最优参数含运气成分。只有当一组参数在**多数股票**上都通过 walk-forward 验证，才具备跨股票泛化能力。

- 取稳健子集（测试盈利 ∧ 过拟合 < 0.4 ∧ 交易 ≥ 5 笔）。
- 对每个参数维度取**中位数**（抗极值），得到「大多数股票都适配」的全局参数。
- 计算**稳定率** = 稳健子集股票数 / 参与股票数。稳定率越高，策略越普适、越稳健。
- 结果存入 `global_strategy_optima`，作为推荐引擎的兜底参数（无 per-stock 参数的股票用全局参数投票）。

### 贝叶斯可信度融合（Learn 环 —— 真实反馈）
推荐结算的真实收益，按投票归因分配到参与策略，融合为策略可信度：

```
后验得分 = 真实胜率 × 真实收益因子
融合权重 w = 真实样本数 / (真实样本数 + 先验强度20)
blendedCredibility = 先验·(1−w) + 后验·w
```

- 样本不足时退化为回测先验；样本充足后逐步信任真实表现（贝叶斯收缩估计）。
- 真实表现好的策略可信度上升、表现差的下降 → 自动淘汰劣质策略（如 RSI 反转稳定率仅 2%，可信度最低）。
- 可信度反哺推荐：共识投票时可信度高的策略发言权更大。

## 数据模型

| 表 | 作用 |
|---|---|
| `strategy_optima` | 每股每策略的 walk-forward 最优参数（per-stock） |
| `global_strategy_optima` | 全局稳健策略（中位数聚合，「最稳定策略」） |
| `strategy_credibility` | 策略可信度（先验回测 + 后验真实融合） |
| `recommendations` | 每日推荐（含 `strategy_detail` 投票归因） |
| `research_runs` | 优化运行历史 |

## API（`/api/research`）

| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/optimize` | 批量 walk-forward 参数优化（后台异步，完成自动聚合+刷新可信度） |
| POST | `/aggregate-global` | 手动提炼全局稳健策略 |
| POST | `/recommend` | 生成今日多策略共识推荐（全局兜底 + 可信度加权） |
| POST | `/resolve` | 结算历史推荐 + 刷新可信度 |
| POST | `/auto-cycle` | 一键日常闭环（聚合→推荐→结算→学习） |
| GET | `/status` `/optima` `/global-optima` `/credibility` `/recommendations` `/performance` `/runs` | 查询 |

## 自动调度

- **生产**：`worker.ts` 的 cron（频率见 `wrangler.toml`）每次触发 `runResearchCycle`，自动聚合、生成推荐、结算、刷新可信度。推荐每日去重、结算幂等。
- **本地**：`RESEARCH_AUTO=1 pnpm dev` 开启自转（启动一次 + 每 6 小时一次）。
- **重优化**（CPU 密集）仍由前端「启动全量策略优化」手动触发，避免阻塞。

## 当前调优结论（基于 2023-02 至 2026-06 共 3.3 年、132 只沪深300成分股、全量 maxSamples=25 优化）

| 策略 | 稳定率 | 平均测试收益 | 夏普 | 覆盖 | 评价 |
|---|---|---|---|---|---|
| MA520战法 | **79%** | 16.4% | 0.52 | 48/61 | 稳健首选 |
| MACD金叉 | **77%** | 14.4% | 0.46 | 44/57 | 最普适 |
| 三周期共振 | 53% | **24.6%** | 0.56 | 25/47 | 高收益、更挑剔 |
| RSI反转 | 4% | 7.6% | 0.27 | 2/47 | 当前市场失效，自动边缘化 |

train/test 收益高度一致（如 MA520 训练 15.6% ≈ 测试 14.4%），证明防过拟合有效。结论：**MA520 与 MACD 金叉组合**（稳定率均 ≥77%）作为平台稳健选股基线，三周期共振（24.6% 收益）作为高弹性增强，RSI 反转在当前市场环境下自动降权淘汰。
